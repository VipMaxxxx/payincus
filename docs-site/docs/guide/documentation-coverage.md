# 文档覆盖能力

这页用于判断当前文档站是否能支撑 PayIncus 作为开源程序对外发布、安装、二次开发和运营接手。

## 当前结论

当前文档已经达到“开源可发布、可部署、可理解主要能力、可开始第三方扩展开发”的水平，但还不是“每个后台字段、每个按钮、每个异常都完整解释”的全量运营手册。

换句话说：开发者可以根据文档理解 PayIncus 的架构、部署、核心功能、扩展中心、主题系统、OAuth、Public API 和 SDK；运维人员可以根据文档完成标准非 Docker 分离部署和生产验收；但深度排障和所有后台细节仍需要继续补齐。

## 与当前开源程序的匹配度

| 能力 | 是否能覆盖当前程序 | 说明 |
| --- | --- | --- |
| 新用户理解项目 | 能 | 首页、项目介绍和功能页已经覆盖核心定位和角色。 |
| 从零部署 | 基本能 | 非 Docker 标准路径、Nginx 分离部署、systemd 和环境变量已覆盖。 |
| 生产验收 | 基本能 | 有 checklist 和验证命令，但真实支付、Incus、SMTP、Lsky、通知仍需要部署者自己补 live proof。 |
| 第三方开发 | 能开始 | 扩展中心、主题系统、Public API、OAuth 和 SDK 已有标准入口。 |
| API 对接 | 基本能 | 当前已有 OpenAPI 3.1、分组 endpoint 文档、scope 和边界说明。 |
| 后台完整运营手册 | 未完全 | 后台每个页面、每个字段、每个异常状态还没有逐项手册。 |
| 故障排查百科 | 未完全 | 只有常见问题和关键链路，未覆盖所有 provider 和宿主机异常组合。 |

## 仍需补齐

- 后台每个配置页的字段说明、默认值、风险等级和修改后影响范围。
- 用户端完整截图式操作流程，包括购买、续费、重装、工单、通知和余额。
- 支付 provider、SMTP、Telegram、Lsky、Incus、Agent 的分场景排障手册。
- 扩展中心真实第三方开发案例，例如“秒杀扩展”的前台页面、后台设置、action、事件和市场发布全过程。
- 主题系统真实主题包示例，包括文件结构、模板槽覆盖、配置项、预览图和市场投稿。
- Admin/User 内部 API 的完整开发者文档；当前公开稳定契约以 Public API 为准。

## 维护规则

每次新增业务能力、公开 API、扩展 capability、主题 slot、环境变量或生产验收脚本，都应同步更新文档站。涉及认证、支付、权限、资源交付、余额和第三方扩展的变更，必须同时更新边界说明和验证方法。
